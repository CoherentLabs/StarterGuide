## Canvas Panel vs. DOM Layouts

UMG's **Canvas Panel** uses absolute anchors to position widgets at fixed points on screen. Gameface uses the **CSS Box Model** and **Flexbox** for layout - a fundamentally responsive system where elements flow relative to their container.

**Key mental shift:** Stop thinking in absolute X/Y coordinates for every widget. Use flex containers for structural layout, and reserve `position: absolute` only for HUD overlays that genuinely need to float at a specific screen corner.

| UMG concept | Gameface equivalent |
|---|---|
| Canvas Panel (absolute anchors) | `position: absolute` + CSS `top/left/right/bottom` |
| Horizontal/Vertical Box | `display: flex` with `flex-direction: row` or `column` |
| Size Box | Explicit `width` / `height` or `flex-basis` |
| DPI Scale rule | Root `font-size` rem scaling (see [Building Scalable & Responsive Game UIs](/layout-assets-and-styling/styling--scalability/building-responsive-game-ui/)) |

## Widget Blueprints vs. JavaScript / TypeScript

UMG **Widget Blueprints** combine visual layout with event-graph logic. In Gameface, these two concerns are cleanly separated:

- **Layout** → HTML + CSS (and optionally a SolidJS/React `.tsx` component for reusable widgets)
- **Logic** → TypeScript / JavaScript (event handlers, state management, engine communication)

This separation makes it easy for artists to work in HTML/CSS while engineers handle the logic layer, without touching the same files.

## Data Binding vs. Property Bindings & Tick Events

UMG uses property bindings (which poll a function every tick) or delegates to sync data. Gameface uses a **push-based data-binding model** - the engine writes to a named model object, and the HTML automatically reflects the change without any polling.

This approach is significantly more performant because updates only happen when data actually changes, not every frame.

See [Core Concepts & Information Flow](/getting-started/core-concepts-and-information-flow/) for a full explanation of the push model, and [Data-Binding Basics](/logic-and-interactions/data-binding--ui-state/data-binding-basics/) for implementation.
